Lesson 8: Probabilitas Bayes

Teorema Bayes, Multi-Evidence, Conditional Independence — dari prior ke posterior

1. Konsep Dasar Probabilitas Bayes

Probabilitas Bayes

Pendekatan berbasis teori statistika klasik untuk menghitung peluang kebenaran suatu hipotesis setelah diperoleh bukti tertentu. Berbeda dengan CF yang berbasis heuristik pakar, Bayes menggunakan perhitungan peluang matematis secara ketat.

1.1 Rumus Teorema Bayes

p(Hi | E) = p(E | Hi) x p(Hi)
———————————————
Σk=1m p(E | Hk) x p(Hk)

1.2 Tiga Jenis Probabilitas

IstilahNotasiMaknaSumber
Prior Probability p(Hi) Probabilitas awal hipotesis Hi sebelum ada bukti Pakar / data historis
Conditional Probability p(E | Hi) Peluang munculnya bukti E jika hipotesis Hi benar Pakar / penelitian
Posterior Probability p(Hi | E) Probabilitas Hi benar setelah melihat bukti E Hasil perhitungan Bayes
Syarat Mutlak: Mutually Exclusive

Pendekatan Bayes mensyaratkan semua hipotesis bersifat saling lepas (mutually exclusive) — tidak boleh ada dua hipotesis yang terjadi bersamaan. Jika H1 terjadi, maka H2 dan H3 tidak mungkin terjadi.

2. Multi-Evidence: Conditional Independence

Ketika ada banyak bukti (E1, E2, ..., En) yang memengaruhi satu hipotesis, perhitungan Bayes menjadi kompleks. Solusinya: Asumsi Kemandirian Bersyarat (Conditional Independence).

Conditional Independence

Setiap observasi atau bukti dianggap berdiri sendiri dan tidak saling memengaruhi satu sama lain dalam membuktikan hipotesis Hi.

Rumus Bayes Multi-Evidence

p(Hi | E1 E2 ... En) = p(E1|Hi) x p(E2|Hi) x ... x p(En|Hi) x p(Hi)
——————————————————————————————
Σk=1m p(E1|Hk) x p(E2|Hk) x ... x p(En|Hk) x p(Hk)

3. Studi Kasus: Iterasi Bayes (3 Hipotesis, 3 Evidence)

Mari kita ikuti studi kasus lengkap. Sistem pakar memiliki 3 hipotesis (H1, H2, H3) dan 3 bukti potensial (E1, E2, E3).

3.1 Tabel Probabilitas Awal (dari Pakar)

ParameterH1H2H3
p(Hi) — Prior0,400,350,25
p(E1 | Hi)0,30,80,5
p(E2 | Hi)0,90,00,7
p(E3 | Hi)0,60,70,9

3.2 Status Awal (Prior State) — Sebelum Ada Bukti

H1: 40%
H2: 35%
H3: 25%

Ini adalah status awal. Sekarang kita amati bagaimana probabilitas bergeser secara dinamis setelah bukti dimasukkan satu per satu.

3.3 Iterasi 1: Pengguna Mengobservasi E3

Langkah 1: Hitung Faktor Normalisasi (Penyebut)

Penyebut = Σ p(E3 | Hk) x p(Hk)
= (0,6 x 0,40) + (0,7 x 0,35) + (0,9 x 0,25)
= 0,24 + 0,245 + 0,225 = 0,71

Langkah 2: Hitung Posterior untuk Masing-masing Hipotesis

p(H1 | E3) = (0,6 x 0,40) / 0,71 = 0,24 / 0,71 = 0,338 → 34%
p(H2 | E3) = (0,7 x 0,35) / 0,71 = 0,245 / 0,71 = 0,345 → 34%
p(H3 | E3) = (0,9 x 0,25) / 0,71 = 0,225 / 0,71 = 0,317 → 32%

Analisis Iterasi 1: Masuknya bukti E3 sangat menguntungkan H3 (naik 7%), sementara H1 dan H2 mulai menurun. Persaingan masih ketat.

3.4 Iterasi 2: Memasukkan E1 (Setelah E3)

Sekarang pengguna memasukkan bukti E1. Perhatikan bahwa kita gunakan perkalian karena asumsi Conditional Independence.

Langkah 1: Hitung Faktor Normalisasi Baru

Penyebut = Σ p(E1|Hk) x p(E3|Hk) x p(Hk)
= (0,3 x 0,6 x 0,40) + (0,8 x 0,7 x 0,35) + (0,5 x 0,9 x 0,25)
= 0,072 + 0,196 + 0,1125 = 0,3805

Langkah 2: Hitung Posterior Terkini

p(H1 | E1 E3) = 0,072 / 0,3805 = 0,189 → 19%
p(H2 | E1 E3) = 0,196 / 0,3805 = 0,515 → 52%
p(H3 | E1 E3) = 0,1125 / 0,3805 = 0,296 → 29%

Progress Bar setelah E3 + E1:

H1: 19%
H2: 52%

Analisis Iterasi 2: Evidence 1 (E1) menghancurkan peluang H1 (turun ke 19%) dan menempatkan H2 sebagai kandidat terkuat (52%). H3 bertahan di 29%.

3.5 Iterasi 3: Memasukkan E2 — The Final Blow

Terakhir, pengguna memasukkan bukti E2. Perhatikan bahwa p(E2 | H2) = 0,0 — artinya bukti E2 tidak mungkin muncul jika H2 benar.

Langkah 1: Hitung Faktor Normalisasi

Penyebut = Σ p(E1|Hk) x p(E2|Hk) x p(E3|Hk) x p(Hk)
= (0,3 x 0,9 x 0,6 x 0,40) + (0,8 x 0,0 x 0,7 x 0,35) + (0,5 x 0,7 x 0,9 x 0,25)
= 0,0648 + 0 + 0,07875 = 0,14355

Langkah 2: Hitung Posterior Akhir

p(H1 | E1 E2 E3) = 0,0648 / 0,14355 = 0,451 → 45%
p(H2 | E1 E2 E3) = 0 / 0,14355 = 0,000 → 0% (ELIMINATED!)
p(H3 | E1 E2 E3) = 0,07875 / 0,14355 = 0,549 → 55%

Progress Bar Final (setelah semua evidence):

H1: 45%
H3: 55%
HipotesisPrior+E3+E3+E1+E3+E1+E2

Critical Insight: Karena p(E2 | H2) = 0,0, maka masuknya E2 seketika mengeliminasi H2 sepenuhnya. Ini adalah kekuatan Teorema Bayes — satu bukti yang mustahil di bawah suatu hipotesis langsung mendiskualifikasi hipotesis tersebut.
Kesimpulan Akhir: Dengan terkumpulnya bukti lengkap (E3, E1, E2), sistem menyimpulkan bahwa H3 adalah hasil diagnosis paling mungkin (55%), diikuti H1 (45%), dan H2 telah tereliminasi.

4. Perbandingan: Certainty Factor vs Probabilitas Bayes

Certainty Factor

  • Landasan: Heuristik & intuisi pakar
  • Asumsi: Fleksibel, hipotesis bisa ganda
  • Kompleksitas: Ringan, linier terhadap jumlah rule
  • Cocok untuk: Diagnosis medis/kerusakan dengan pakar dominan

Probabilitas Bayes

  • Landasan: Statistika formal, kalkulasi ketat
  • Asumsi: Mutually Exclusive + Conditional Independence
  • Kompleksitas: Eksponensial dengan jumlah hipotesis/bukti
  • Cocok untuk: Klasifikasi dengan data historis berlimpah

5. Rangkuman

Tiga Poin Kunci Bayes
  1. Prior → Posterior: Bayes memperbarui keyakinan awal (prior) dengan bukti baru untuk menghasilkan keyakinan akhir (posterior).
  2. Mutually Exclusive: Hipotesis harus saling lepas — hanya satu yang bisa benar.
  3. Conditional Independence: Asumsi bahwa bukti-bukti tidak saling memengaruhi, sehingga probabilitas bisa dikalikan.
