Lesson 7: Certainty Factor Deep Dive

Perhitungan detail CF, MB/MD, CF Gabungan, Resolusi Konflik — dari teori ke studi kasus

1. Sumber Ketidakpastian dalam Sistem Pakar

Sebelum masuk ke rumus, penting memahami mengapa sistem pakar butuh mekanisme ketidakpastian. Ada dua sumber utama keraguan:

1. Ketidakpastian Data (Input Uncertainty)
  • Fakta observasi di lapangan tidak lengkap / tidak utuh
  • Data dari sumber dengan tingkat kepercayaan rendah
  • Data dari berbagai sumber yang saling berkonflik
2. Ketidakpastian Inferensi (Rule Uncertainty)
  • Aturan dari pakar bersifat heuristik (berdasarkan pengalaman praktis), bukan rumus pasti
  • Aturan didesain general, sering gagal akomodasi kasus anomali spesifik

Kesimpulan: Sistem berbasis aturan butuh mekanisme matematika untuk mengubah inexact reasoning menjadi kesimpulan kuantitatif yang presisi.

2. Formula Dasar Certainty Factor

Definisi: Certainty Factor (CF)

Certainty Factor mengukur tingkat keyakinan terhadap suatu hipotesis berdasarkan bukti yang ada. Diperkenalkan oleh Shortliffe & Buchanan (1975) pada sistem MYCIN (diagnosis medis).

2.1 Rumus Utama

CF(H, E) = MB(H, E) − MD(H, E)
KomponenMaknaRentang
CF(H, E) Certainty Factor hipotesis H berdasarkan bukti E -1.0 (pasti salah) – 1.0 (pasti benar)
MB(H, E) Measure of Belief — ukuran kepercayaan pada H karena E 0 – 1
MD(H, E) Measure of Disbelief — ukuran ketidakpercayaan pada H karena E 0 – 1

2.2 Rumus Detil MB dan MD

MB(H, E) = ( max[P(H|E), P(H)] - P(H) ) / ( 1 - P(H) )
Jika P(H) = 1, maka MB(H, E) = 1
MD(H, E) = ( P(H) - min[P(H|E), P(H)] ) / P(H)
Jika P(H) = 0, maka MD(H, E) = 1

Dimana:

  • P(H) = Probabilitas awal (prior) kebenaran hipotesis H
  • P(H|E) = Probabilitas bersyarat bahwa H benar jika bukti E terjadi

3. Contoh Perhitungan: Diagnosis Phimosis

Soal:

Seorang pakar penyakit dalam menyatakan bahwa probabilitas seseorang terkena penyakit Phimosis secara umum adalah 0,02. Data lapangan menunjukkan dari 100 orang dengan gejala kulit berminyak, 40 orang positif Phimosis. Hitung Certainty Factor bahwa Phimosis disebabkan oleh kulit berminyak.

Pemetaan Variabel:

  • H = Terjadinya penyakit Phimosis
  • E = Gejala kulit berminyak
  • P(H) = 0,02
  • P(H|E) = 40/100 = 0,4

Langkah 1: Hitung MB(H, E)

Karena P(H|E) > P(H), maka MB dihitung:

MB(H, E) = (0,4 − 0,02) / (1 − 0,02)
= 0,38 / 0,98 = 0,3878

Langkah 2: Hitung MD(H, E)

Karena P(H|E) >= P(H) (bukti E meningkatkan kepercayaan), maka:

MD(H, E) = 0

Langkah 3: Hitung CF(H, E)

CF = 0,3878 − 0 = 0,3878

Interpretasi: Terdapat kecenderungan keyakinan yang kuat pada Belief (kepercayaan), dengan nilai CF positif 0,3878 mendekati 0,4.

4. Linguistic-to-Numeric Mapping

Pakar manusia menggunakan istilah kualitatif. Sistem pakar memetakannya ke skala numerik:

Istilah LinguistikNilai CF
Pasti Tidak (Definitely not)-1,0
Kemungkinan Besar Tidak (Probably not)-0,6
Tidak Tahu (Unknown)0,0
Kemungkinan Besar (Probably)0,6
Hampir Pasti (Almost certainly)0,8
Pasti (Definitely)1,0
Contoh Translasi:
Pakar: "Jika pasien mengalami batuk dan panas, maka hampir dipastikan penyakitnya influenza."
Mesin: IF batuk AND panas THEN influenza (CF = 0,8)

5. Perhitungan CF Gabungan

Dalam sistem pakar nyata, satu aturan sering dipicu oleh beberapa bukti sekaligus. Ada tiga skenario utama:

5.1 Skenario 1: Evidence Tunggal, Hipotesis Tunggal

IF E THEN H (CF Rule)
CF(H, E) = CF(E) x CF(Rule)
Catatan Praktis:
  • CF(Rule) bersifat statis — ditentukan pakar saat sistem dibangun
  • CF(E) bersifat dinamis — dimasukkan pengguna saat konsultasi
Contoh:
Rule: IF hari ini terang THEN besok hujan (CF Rule = 0,6)
User: CF(hari ini terang) = 0,4
CF(besok hujan, hari ini terang) = 0,4 x 0,6 = 0,24

5.2 Skenario 2: Evidence Ganda, Hipotesis Tunggal

Logika AND (Konjungsi): Rantai AND sekuat mata rantai terlemahnya.

IF E1 AND E2 AND ... En THEN H (CF Rule)
CF(H, E) = min[ CF(E1), CF(E2), ..., CF(En) ] x CF(Rule)

Logika OR (Disjungsi): Ambil indikator tunggal yang paling kuat.

IF E1 OR E2 OR ... En THEN H (CF Rule)
CF(H, E) = max[ CF(E1), CF(E2), ..., CF(En) ] x CF(Rule)
Contoh AND:
Rule: IF demam AND batuk AND muntah THEN penyakit = TBC (CF Rule = 0,3)
CF(demam)=0,4; CF(batuk)=0,2; CF(muntah)=0,7
CF = min(0,4; 0,2; 0,7) x 0,3 = 0,2 x 0,3 = 0,06

6. Resolusi Konflik: Dua Aturan, Satu Hipotesis

Kasus klasik: Ada dua aturan berbeda (dengan bukti E1 dan E2) yang menghasilkan kesimpulan yang sama (H).

Rule 1: IF E1 THEN H (CF Rule 1)
CF1 = CF(E1) x CF(Rule 1)

Rule 2: IF E2 THEN H (CF Rule 2)
CF2 = CF(E2) x CF(Rule 2)

Tiga Kondisi Resolusi

Kondisi 1: Saling Memperkuat (CF1 >= 0 dan CF2 >= 0)

CFcombine = CF1 + CF2 x (1 − CF1)

Kondisi 2: Saling Bertentangan (satu positif, satu negatif)

CFcombine = (CF1 + CF2) / (1 − min[ |CF1|, |CF2| ])

Kondisi 3: Saling Melemahkan (CF1 < 0 dan CF2 < 0)

CFcombine = CF1 + CF2 x (1 + CF1)

Contoh: Diagnosis Batuk Rejan

Rule 1: IF batuk THEN batuk rejan (CF = 0,8)
Rule 2: IF demam THEN batuk rejan (CF = 0,6)

Kasus A: Kedua gejala positif
CF(batuk)=1, CF(demam)=1

CF1 = 1 x 0,8 = 0,8
CF2 = 1 x 0,6 = 0,6
Kedua ≥ 0 ⇒ CF = 0,8 + 0,6(1-0,8) = 0,8 + 0,12 = 0,92

Kasus B: Satu positif, satu negatif
CF(batuk)=1, CF(demam)=-1

CF1 = 0,8; CF2 = -0,6
Berbeda tanda ⇒ CF = (0,8 - 0,6) / (1 - min[|0,8|, |-0,6|])
= 0,2 / (1 - 0,6) = 0,2 / 0,4 = 0,50

Kasus C: Kedua gejala negatif
CF(batuk)=-1, CF(demam)=-1

CF1 = -0,8; CF2 = -0,6
Kedua < 0 ⇒ CF = -0,8 + (-0,6)(1 + (-0,8))
= -0,8 - 0,12 = -0,92

7. Studi Kasus: Sistem FORECAST

Sistem FORECAST adalah aplikasi prakiraan cuaca dengan 6 aturan pakar. Mari kita analisis langkah demi langkah.

Basis Pengetahuan (6 Rules)

R1: IF today is rain THEN tomorrow is rain (CF = 0,5)
R2: IF today is dry THEN tomorrow is dry (CF = 0,5)
R3: IF today is rain AND rainfall is low THEN tomorrow is dry (CF = 0,6)
R4: IF today is rain AND rainfall is low AND temperature is cold THEN tomorrow is dry (CF = 0,7)
R5: IF today is dry AND temperature is warm THEN tomorrow is rain (CF = 0,65)
R6: IF today is dry AND temperature is warm AND sky is overcast THEN tomorrow is rain (CF = 0,55)

Input Pengguna

FaktaCF
Rain Today1,0 (pasti hujan)
Rainfall Low0,8 (yakin rendah)
Cold0,9 (yakin dingin)

Langkah 1: Evaluasi Aturan yang Terpicu

R1 terpicu (rain hari ini = 1,0):

CF_R1 = 1,0 x 0,5 = 0,50 (Prediksi: Hujan)

R3 terpicu (rain=1,0 AND rainfall low=0,8):

CF_R3 = min(1,0; 0,8) x 0,6 = 0,8 x 0,6 = 0,48 (Prediksi: Kering)

R4 terpicu (rain=1,0 AND rainfall low=0,8 AND cold=0,9):

CF_R4 = min(1,0; 0,8; 0,9) x 0,7 = 0,8 x 0,7 = 0,56 (Prediksi: Kering)

Langkah 2: Resolusi Konflik untuk Prediksi "Kering"

R3 dan R4 sama-sama memprediksi "Kering". Keduanya positif, gunakan rumus saling memperkuat:

CF_combine = 0,48 + 0,56 x (1 − 0,48)
= 0,48 + 0,56 x 0,52
= 0,48 + 0,2912 = 0,7712 (dibulatkan 0,77 untuk Kering)

Langkah 3: Keputusan Akhir

PrediksiCF
Esok Kering0,77
Esok Hujan0,50
Kesimpulan: Sistem memutuskan "Esok Hari Kering" karena CF 0,77 mengalahkan prediksi hujan 0,50. Meskipun hari ini hujan, kombinasi curah hujan rendah + suhu dingin memberikan keyakinan lebih kuat bahwa besok akan kering.
