Lesson 17: Arsitektur JST & Fungsi Aktivasi

3 penentu perilaku JST, klasifikasi arsitektur (single/multi, feedforward/recurrent), dan 4 fungsi aktivasi

1. Tiga Penentu Perilaku JST

Kemampuan dan perilaku sebuah JST sangat ditentukan oleh tiga hal utama. Pahami ketiganya sebelum masuk ke detail arsitektur:

  1. Pola Hubungan Antar Neuron (Arsitektur Jaringan) — menentukan topologi: single vs multilayer, feedforward vs recurrent.
  2. Fungsi Aktivasi Neuron — menentukan transformasi net input menjadi output (linear/non-linear, range output).
  3. Metode Pelatihan (Training/Learning) — menentukan algoritma untuk mencari bobot optimal (Hebb, Perceptron, Backprop, dll).
Inti Konsep

Ketiga faktor ini terkait erat: arsitektur menentukan "bentuk" jaringan, aktivasi menentukan "jenis output" yang bisa dihasilkan, dan training menentukan "kualitas" bobot akhir. Mengubah salah satu = perilaku JST berubah drastis.

2. Struktur Lapisan (Layer) JST

LayerFungsiPenentuan Jumlah Node
Input Layer Menerima masukan dari luar berupa fitur-fitur data. Sesuai dengan jumlah fitur data. Misal data dengan 2 fitur (Mass, Speed) → 2 node input.
Hidden Layer Berada di antara input dan output; melakukan transformasi non-linear. Tidak ada rumus pasti. Ditentukan melalui eksperimen (by experiment).
Output Layer Menghasilkan prediksi akhir kelas atau nilai. Sesuai dengan jenis keluaran target. Binary classification → 1 node output; 10-kelas → 10 node.

3. Klasifikasi Arsitektur JST

3.1 Berdasarkan Jumlah Lapisan

Single-Layer Network

Hanya memiliki satu lapisan bobot penghubung — langsung dari input ke output. Tidak ada hidden layer. Contoh klasik: Perceptron (1958), HebbNet.

Multilayer Network

Memiliki satu atau lebih hidden layer di antara input dan output. Mampu menyelesaikan masalah non-linear (mis. XOR). Contoh: Backpropagation network (1986+).

3.2 Berdasarkan Aliran Sinyal

JenisKarakteristik AliranContoh
Feedforward Aliran sinyal berjalan searah dari input → output, tanpa loop/umpan balik. Perceptron, HebbNet, McCulloch-Pitts, Backpropagation
Recurrent Memiliki jalur umpan balik (feedback loops) — output dari suatu neuron bisa kembali menjadi input bagi neuron di lapisan sebelumnya atau sama. Hopfield Network (1982), RNN modern, LSTM
Perhatikan untuk UAS

Yang kita bahas di PDF ini (MCP, HebbNet, Perceptron) semuanya adalah Single-Layer Feedforward. Kelemahan utama single-layer: tidak bisa menyelesaikan masalah non-linear (akan dibuktikan di Lesson 18 pada kasus XOR).

4. Net Input dan Output Neuron

Setelah arsitektur, komponen penting berikutnya adalah cara neuron memproses input menjadi output. Ada dua langkah:

1. Net Input (z) — jumlah berbobot (sum of product):

z = x1·w1 + x2·w2 + ... + xn·wn
z = Σi=1n xi · wi

2. Output (y) — masukkan net input ke fungsi aktivasi f:

y = f(z)
Contoh Numerik

Misal: x1=2, w1=0,5; x2=3, w2=0,3; x3=1, w3=0,2.

Net input:

z = (2)(0,5) + (3)(0,3) + (1)(0,2) = 1,0 + 0,9 + 0,2 = 2,1

Maka output y = f(2,1). Nilai f(2,1) tergantung jenis fungsi aktivasi yang dipilih (lihat bagian 5).

5. Empat Fungsi Aktivasi Populer

Fungsi aktivasi menentukan bagaimana z ditransformasi menjadi y. Empat yang paling sering dipakai:

Gambar 1. Binary Step: y=0 jika x<0, y=1 jika x≥0.
Gambar 2. Sigmoid: y=1/(1+e^(-x)), output (0,1).
Gambar 3. tanh: y=(e^x - e^(-x))/(e^x + e^(-x)), output (-1,1).
Gambar 4. ReLU: y=max(0, x), output [0, ∞). Garis y=x berlanjut ke atas untuk x>1 (di luar area plot).
FungsiFormulaRange OutputKarakteristik
Binary Step y = 1 jika z ≥ 0; 0 jika z < 0 {0, 1} Sederhana, non-diferentiable di z=0. Cocok untuk klasifikasi biner.
Sigmoid / Logistic σ(x) = 1 / (1 + e-x) (0, 1) Smooth (diferentiable), S-shape. Masalah: vanishing gradient untuk |x| besar.
tanh tanh(x) = (ex - e-x) / (ex + e-x) (-1, 1) Sama seperti sigmoid tapi zero-centered. Versi PDF: tanh(x/2) = (1-e-x)/(1+e-x).
ReLU ReLU(x) = max(0, x) [0, ∞) Komputasi cepat, tidak ada vanishing gradient untuk x>0. Default untuk hidden layer modern.
Catatan Formula tanh di PDF

PDF menuliskan tanh(x/2) = (1 - e-x) / (1 + e-x). Bentuk ini ekuivalen dengan tanh standar. Untuk konsistensi dengan notasi umum, kita pakai tanh(x) = (ex - e-x) / (ex + e-x).

6. Contoh Aplikasi: Klasifikasi Jenis Pesawat

Mari lihat aplikasi konkret. Misalkan kita ingin mengklasifikasikan jenis pesawat terbang ke kelas Bomber (label +1) atau Fighter (label -1):

Aspek Spesifikasi
Task (T) Klasifikasi jenis pesawat → Bomber (+1) atau Fighter (-1)
Fitur Input Mass (x1) dan Speed (x2) — maka 2 node input
Output (y) 1 node output untuk menghasilkan prediksi kelas (+1 atau -1)
Training Cari kombinasi bobot w1 dan w2 yang memisahkan data Bomber dan Fighter secara akurat

Setelah training, JST akan memiliki bobot (w1, w2) yang merepresentasikan "garis pemisah" terbaik antara kedua kelas. Inilah yang disebut decision boundary — detail formula akan dibahas di Lesson 18.
