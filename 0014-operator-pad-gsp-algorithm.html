Lesson 14: Operator PAD & GSP

Empat operator lengan robot (STACK/UNSTACK/PICKUP/PUTDOWN) + algoritma Goal Stack Planning

1. Mengapa Perlu Operator Formal?

Definisi: Operator PAD

Setiap aksi lengan robot didefinisikan formal dengan 3 elemen logika:

  • P (Precondition) — Syarat yang HARUS TRUE sebelum aksi dapat dijalankan.
  • A (Add list) — Fakta baru yang menjadi TRUE setelah aksi selesai.
  • D (Delete list) — Fakta lama yang menjadi FALSE setelah aksi selesai.

Dengan PAD, sebuah operator sepenuhnya deskriptif — bisa di-stack, di-evaluate, dan di-combine oleh algoritma. Untuk Blocks World, ada tepat 4 operator.

2. Empat Operator Lengan Robot

2.1 STACK(x, y) — Tumpuk x ke atas y

STACK(x, y)

Meletakkan balok x yang sedang dipegang ke atas balok y.

P (Pre): CLEAR(y) ∧ HOLDING(x)
A (Add): ON(x,y) ∧ ARMEMPTY
D (Del): HOLDING(x) ∧ CLEAR(y)

2.2 UNSTACK(x, y) — Ambil x dari atas y

UNSTACK(x, y)

Mengambil/mengangkat balok x yang berada di atas balok y.

P (Pre): ON(x,y) ∧ CLEAR(x) ∧ ARMEMPTY
A (Add): HOLDING(x) ∧ CLEAR(y)
D (Del): ON(x,y) ∧ ARMEMPTY

2.3 PICKUP(x) — Ambil x dari meja

PICKUP(x)

Mengangkat balok x dari permukaan meja.

P (Pre): ONTABLE(x) ∧ CLEAR(x) ∧ ARMEMPTY
A (Add): HOLDING(x)
D (Del): ONTABLE(x) ∧ ARMEMPTY

2.4 PUTDOWN(x) — Letakkan x ke meja

PUTDOWN(x)

Meletakkan balok x yang sedang dipegang ke permukaan meja.

P (Pre): HOLDING(x)
A (Add): ONTABLE(x) ∧ ARMEMPTY
D (Del): HOLDING(x)

2.5 Ringkasan dalam Satu Tabel

Operator Aksi Fisik P (Precondition) A (Add) D (Delete)
STACK(x,y) Tumpuk x ke y CLEAR(y) ∧ HOLDING(x) ON(x,y) ∧ ARMEMPTY HOLDING(x) ∧ CLEAR(y)
UNSTACK(x,y) Ambil x dari y ON(x,y) ∧ CLEAR(x) ∧ ARMEMPTY HOLDING(x) ∧ CLEAR(y) ON(x,y) ∧ ARMEMPTY
PICKUP(x) Ambil x dari meja ONTABLE(x) ∧ CLEAR(x) ∧ ARMEMPTY HOLDING(x) ONTABLE(x) ∧ ARMEMPTY
PUTDOWN(x) Taruh x ke meja HOLDING(x) ONTABLE(x) ∧ ARMEMPTY HOLDING(x)
Pattern ARMEMPTY

Setiap operator yang menggunakan/melepas lengan robot memiliki ARMEMPTY di P atau A/D list. Kenapa? Karena lengan hanya bisa pegang 1 balok. ARMEMPTY jadi "latch" yang memastikan kondisi lengan konsisten.

3. Goal Stack Planning (GSP)

Definisi: Goal Stack Planning (GSP)

Algoritma planning klasik yang menggunakan struktur data Stack (LIFO) untuk menentukan urutan aksi. Memecah Goal State menjadi sub-goal, lalu memilih operator yang bisa memenuhi sub-goal tersebut, satu per satu dari atas stack.

3.1 Komponen GSP

Komponen Fungsi
Stack Tumpukan memori untuk sub-goal dan operator yang sedang dievaluasi. LIFO (Last In, First Out).
Current-state Database fakta yang mencatat kondisi dunia balok saat ini. Update setelah setiap aksi.
Daftar-PAD Kumpulan 4 operator robot (STACK, UNSTACK, PICKUP, PUTDOWN) dengan efek logikanya.
Queue (Rencana Solusi) Antrean linear tempat menampung urutan aksi final yang berhasil dieksekusi.

3.2 Algoritma GSP (3 Step Loop)

1
Inisialisasi Stack

Tempatkan seluruh kondisi tujuan akhir (Goal State) sebagai compound goal di posisi stack paling bawah.

2
Pecah Compound Goal

Pecah compound goal, masukkan masing-masing sub-goal tunggal (yang belum terpenuhi) ke slot stack di atasnya.

3
Loop hingga Stack Kosong

Evaluasi elemen teratas stack dengan 4 aturan:

Jika Top of Stack adalah... Action
Fakta/kondisi sudah TRUE di Current-state POP (hapus dari stack)
Sub-goal belum terpenuhi Cari operator yang memiliki sub-goal di Add list. Push operator menggantikan sub-goal, lalu push seluruh prekondisi operator di atasnya.
Operator aksi & semua prekondisi TRUE di Current-state POP operator, masukkan ke Queue (Rencana Solusi), update Current-state dengan Add & Delete list operator.
Current-state == Goal State STOP dengan status Sukses.
Logika Inti GSP

GSP bekerja dengan backward chaining dari Goal ke Initial State: cari operator yang bisa memenuhi sub-goal, lalu buat sub-goal baru berupa prekondisi operator tersebut. Ulangi sampai Current-state sama dengan Goal State.

3.3 Mini-Example: 1 Sub-goal

Ilustrasi: Bagaimana GSP Memilih Operator?

Situasi: Stack saat ini = [ ..., ON(A,B) ].

Evaluasi: Top of stack adalah sub-goal ON(A,B). Cek Current-state: ON(A,B) FALSE. Lihat Daftar-PAD: operator yang Add list-nya mengandung ON(A,B) adalah STACK(A,B).

Aksi GSP: Ganti ON(A,B) di stack dengan STACK(A,B), lalu push prekondisi STACK(A,B) di atasnya.

Stack baru = [ ..., STACK(A,B), CLEAR(B), HOLDING(A) ]

Sekarang algoritma akan evaluasi CLEAR(B), lalu HOLDING(A) sebagai sub-goal baru, dst.

Jebakan: Urutan Stack Penting

Perhatikan urutan push prekondisi di stack: CLEAR(B) masuk DULU (di bawah), HOLDING(A) masuk KEMUDIAN (di atas). Karena LIFO, HOLDING(A) akan dievaluasi DULU. Ini bukan salah — urutan boleh dibalik untuk sub-goal yang tidak saling bergantung, tapi urutan ini adalah default dari PDF.

4. Cheat Matrix: Pilih Operator yang Tepat

Kondisi Awal Tujuan Operator
x di atas y (lengan kosong) x dipegang, y clear UNSTACK(x,y)
x dipegang, y clear (x ≠ meja) x di atas y, lengan kosong STACK(x,y)
x di meja, clear (lengan kosong) x dipegang PICKUP(x)
x dipegang (lengan penuh) x di meja, lengan kosong PUTDOWN(x)
Pasangan Komplementer
  • STACK ⇔ UNSTACK (tumpuk ⇔ ambil dari tumpukan)
  • PICKUP ⇔ PUTDOWN (ambil dari meja ⇔ taruh ke meja)

Sering dalam satu tracing, GSP akan PICKUP lalu STACK (atau UNSTACK lalu PUTDOWN).
