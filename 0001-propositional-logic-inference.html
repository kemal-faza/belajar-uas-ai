Propositional Logic & Inferensi

Lesson 1 dari materi Reasoning KBA — Fondasi logika untuk UAS

Prerequisites: Kamu sudah paham Wumpus World, Knowledge Representation (5 jenis, 4 teknik implementasi), dan definisi dasar logika (syntax vs semantics, propositional vs predicate logic).

1. Recap: Logika Itu Apa?

Logika adalah bahasa formal untuk merepresentasikan kalimat (sentence) yang memungkinkan kita menarik kalimat baru melalui inferensi.

Ada 2 komponen utama:

  • Syntax — aturan yang mendefinisikan kalimat yang sah secara struktur
  • Semantics — aturan yang mendefinisikan makna kalimat atau kebenarannya di dunia nyata

Contoh syntax PL (Propositional Logic):

  • P ∧ Q ✅ valid (dua proposisi dihubungkan ∧)
  • ∧ P Q ❌ tidak valid (operator di posisi salah)

2. Propositional Logic — Syntax

Propositional Logic bekerja dengan proposisi — pernyataan yang bernilai True atau False.

Atomic Sentences (Proposisi Dasar)

  • Simbol proposisi: P, Q, R, W₁,₂, P₃,₁, ...
  • Nilai kebenaran: True, False
  • Contoh: W₁,₃ = "Wumpus ada di [1,3]"

Logical Connectives (Penghubung Logika)

Simbol Nama Baca Contoh
¬ Negation "not" / "tidak" ¬W₁,₃ = Wumpus tidak di [1,3]
∧ Conjunction "and" / "dan" W₁,₃ ∧ P₃,₁ = Wumpus di [1,3] dan Pit di [3,1]
∨ Disjunction "or" / "atau" P₂,₂ ∨ P₃,₁ = Pit di [2,2] atau [3,1]
⇒ Implication "implies" / "jika-maka" B₂,₁ ⇒ (P₁,₁ ∨ P₂,₂ ∨ P₃,₁)
⇔ Biconditional "iff" / "jika dan hanya jika" B₂,₁ ⇔ (P₁,₁ ∨ P₂,₂ ∨ P₃,₁)
Ingat: Di Wumpus World, Breeze artinya ada Pit di kotak tetangga. Makanya B₂,₁ ⇔ (P₁,₁ ∨ P₂,₂ ∨ P₃,₁) — Breeze di [2,1] benar jika dan hanya jika ada Pit di salah satu tetangganya.

3. Propositional Logic — Semantics

Semantics mendefinisikan kapan suatu kalimat bernilai True. Ini ditentukan oleh truth values dari atomic propositions dan aturan logical connectives.

Truth Table (Tabel Kebenaran)

PQ¬PP ∧ QP ∨ QP ⇒ QP ⇔ Q
FFTFFTT
FTTFTTF
TFFFTFF
TTFTTTT
Perhatian khusus untuk UAS: Implication P ⇒ Q hanya False saat P True dan Q False. Sisanya True! Ini sering muncul di soal.

4. Model dan Entailment

  • Model = sebuah "dunia" di mana setiap proposisi punya nilai (True/False). Contoh: model untuk W₁,₃ dan P₃,₁ punya 4 kemungkinan kombinasi.
  • Entailment (⊨) — KB ⊨ α artinya: di semua model dimana Knowledge Base (KB) True, α juga True. Ini adalah fondasi inferensi!
KB ⊨ α ↔ (KB ⇒ α) valid
"Jika KB benar, maka α pasti benar"

5. Inferensi pada PL

Ada 3 pendekatan inferensi:

A. Enumeration (Tabel Kebenaran)

Cara paling sederhana: cek semua kemungkinan model. Cocok untuk 2-3 proposisi, tapi tidak feasible untuk banyak proposisi (2n model).

B. Inference Rules

Aturan-aturan yang memungkinkan kita menarik kesimpulan tanpa enumerasi semua model.

Rule Bentuk Contoh
Modus Ponens α ⇒ β, α ⊢ β "Jika ada Breeze maka ada Pit, ada Breeze → simpulkan: ada Pit"
And Elimination α ∧ β ⊢ α W₁,₃ ∧ P₃,₁ → W₁,₃
And Introduction α, β ⊢ α ∧ β Dari dua fakta terpisah, gabung jadi satu
Or Introduction α ⊢ α ∨ β Dari P₂,₂ bisa simpulkan P₂,₂ ∨ P₃,₁
Unit Resolution α ∨ β, ¬β ⊢ α "Pit di [2,2] atau [3,1], bukan [2,2] → Pit di [3,1]"
Resolution α ∨ β, ¬β ∨ γ ⊢ α ∨ γ Bentuk umum dari Unit Resolution

C. Horn Form & Modus Ponens

Horn clause = clause yang punya paling banyak satu literal positif. Contoh:

  • ¬B₂,₁ ∨ P₁,₁ ∨ P₂,₂ ∨ P₃,₁ — bentuk setara dengan B₂,₁ ⇒ (P₁,₁ ∨ P₂,₂ ∨ P₃,₁)
  • Bisa ditulis sebagai: (P₁,₁ ∧ P₂,₂ ⇒ P₃,₁)

Inferensi dengan Horn Form efisien pakai Forward Chaining atau Backward Chaining — linear time!

D. Resolusi pada CNF

Langkah-langkah:

  1. Ubah semua kalimat ke Conjunctive Normal Form (CNF) — kumpulan clause yang di-∧
  2. Setiap clause adalah disjunction (∨) dari literal
  3. Terapkan resolution rule: (ℓ₁ ∨ ... ∨ ℓᵢ ∨ ... ∨ ℓₙ), (m₁ ∨ ... ∨ ¬ℓᵢ ∨ ... ∨ mₖ) ⊢ (ℓ₁ ∨ ... ∨ ℓₙ ∨ m₁ ∨ ... ∨ mₖ)
  4. Ulang sampai dapat False (kontradiksi) — bukti bahwa KB ⊨ α
CNF vs Horn: Resolusi butuh waktu eksponensial di worst case. Horn Form bisa linear. Tapi tidak semua kalimat bisa diubah ke Horn Form.

6. Studi Kasus: Wumpus World

Misalkan agen di [1,1] tidak mendeteksi apa-apa. Lalu bergerak ke [2,1] dan mendeteksi Breeze.

Knowledge Base:

R₁ : ¬B₁,₁                    (tidak ada Breeze di [1,1])
R₂ : B₁,₁ ⇔ (P₁,₂ ∨ P₂,₁)    (Breeze di [1,1] → Pit di tetangga)
R₃ : ¬P₁,₁                    (tidak ada Pit di start — aturan game)
R₄ : B₂,₁                     (ada Breeze di [2,1])
R₅ : B₂,₁ ⇔ (P₁,₁ ∨ P₂,₂ ∨ P₃,₁)

Inferensi dengan Resolution:

Dari R₅ dan B₂,₁:  P₁,₁ ∨ P₂,₂ ∨ P₃,₁        (Breeze → ada Pit di tetangga)
Dari R₃:            ¬P₁,₁                      (tidak ada Pit di start)
Resolve:            P₂,₂ ∨ P₃,₁                (Pit di [2,2] atau [3,1])

Ini persis seperti reasoning yang kamu lihat di gambar slides sebelumnya!

7. Quiz: Cek Pemahamanmu

Quiz 1: Implication

Kapan P ⇒ Q bernilai False?

Saat P = False, Q = True
Saat P = True, Q = False
Saat P = True, Q = True
Saat P = False, Q = False

Quiz 2: Entailment

Jika KB = {W₁,₃ ⇒ P₂,₂}, apakah KB ⊨ ¬W₁,₃?

Ya, karena W₁,₃ ⇒ P₂,₂ berarti W₁,₃ pasti False
Tidak, karena W₁,₃ bisa True selama P₂,₂ juga True

Quiz 3: Inference Rules

Dari fakta (P₂,₂ ∨ P₃,₁) dan ¬P₂,₂, apa yang bisa kamu simpulkan?

P₂,₂
P₃,₁ ∧ ¬P₂,₂
P₃,₁
Tidak bisa menyimpulkan apa-apa

8. Ringkasan

  • Syntax PL: atomic sentences (P, Q, R) + 5 connectives (¬, ∧, ∨, ⇒, ⇔)
  • Semantics PL: ditentukan oleh truth values + truth tables
  • Entailment (⊨): KB ⊨ α jika α True di semua model dimana KB True
  • Inferensi: tabel kebenaran (eksponensial), inference rules (Modus Ponens, And Elim, Resolution), Horn Form (linear)
