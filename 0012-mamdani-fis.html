Lesson 12: Sistem Inferensi Fuzzy & Mamdani

FIS 4-langkah: Fuzzifikasi → Implikasi → Agregasi → Defuzzifikasi, plus studi kasus produksi lengkap

1. Apa itu FIS?

Definisi: Fuzzy Inference System (FIS)

FIS adalah proses penarikan kesimpulan dari sekumpulan kaidah/aturan fuzzy (minimal 2 aturan) untuk memetakan input tegas (crisp) menjadi output tegas (crisp). Ibarat "memanggang kue" dengan input: bahan → aturan resep → output: kue matang.

Contoh penerapan FIS di industri:

  • Mesin Cuci: Input (massa pakaian, jenis kotoran, kepekaan) → Output (kecepatan putar)
  • Air Conditioner (AC): Input (suhu setting, suhu aktual, titik embun) → Output (kecepatan kompresor, kipas, mode)
  • Optimasi Produksi: Input (permintaan, persediaan) → Output (jumlah produksi)

2. Empat Langkah FIS (Mamdani)

Metode Mamdani (Ebrahim Mamdani, 1975)

Langkah-langkah:

  1. Fuzzifikasi — map nilai input crisp ke derajat keanggotaan fuzzy
  2. Evaluasi aturan & implikasi — hitung α-predikat dengan operator Zadeh, terapkan MIN
  3. Agregasi/Komposisi — gabungkan output semua aturan dengan MAX
  4. Defuzzifikasi — konversi kurva komposit ke satu nilai crisp dengan Centroid (default)

Gambar 1: Alur 4 langkah FIS Mamdani dari input crisp ke keputusan crisp tunggal.

3. Studi Kasus: Optimasi Produksi Makanan Kaleng ABC

Mari kita kerjakan studi kasus lengkap dari PDF. Perusahaan ingin menentukan jumlah produksi (z) berdasarkan permintaan (x) dan persediaan (y).

3.1 Parameter Sistem

VariabelSimbolRange
Permintaan (input)x[1000, 5000] kemasan/hari
Persediaan (input)y[100, 600] kemasan/hari
Produksi (output)z[2000, 7000] kemasan/hari

3.2 Basis Aturan (Rule Base)

[R1] IF Permintaan TURUN AND Persediaan BANYAK THEN Produksi BERKURANG
[R2] IF Permintaan TURUN AND Persediaan SEDIKIT THEN Produksi BERKURANG
[R3] IF Permintaan NAIK AND Persediaan BANYAK THEN Produksi BERTAMBAH
[R4] IF Permintaan NAIK AND Persediaan SEDIKIT THEN Produksi BERTAMBAH

3.3 Fungsi Keanggotaan (Asumsi Linear)

PDF tidak memberikan parameter a, b eksplisit untuk semua himpunan, tapi dari perhitungan yang ditunjukkan, kita bisa merekonstruksi:

  • Permintaan TURUN: linear turun dari 0 di x=1000 ke 1 di x=5000
  • Permintaan NAIK: linear naik dari 0 di x=1000 ke 1 di x=5000
  • Persediaan SEDIKIT: linear turun dari 1 di y=100 ke 0 di y=600
  • Persediaan BANYAK: linear naik dari 0 di y=100 ke 1 di y=600
  • Produksi BERKURANG: linear turun (dari puncak 1 ke 0)
  • Produksi BERTAMBAH: linear naik μ(z) = (z-2000)/5000

3.4 Pertanyaan

Problem

Berapa z (jumlah produksi) yang optimal jika Permintaan = 4000 dan Persediaan = 300?

1

Fuzzifikasi — Hitung Derajat Keanggotaan Input

Untuk setiap himpunan input, hitung μ pada nilai crisp yang diketahui.

1a. Variabel Permintaan (x = 4000, range [1000, 5000])

μPmtTURUN[4000] = (5000 − 4000) / 4000 = 1000/4000 = 0,25

μPmtNAIK[4000] = (4000 − 1000) / 4000 = 3000/4000 = 0,75

1b. Variabel Persediaan (y = 300, range [100, 600])

μPsdSEDIKIT[300] = (600 − 300) / 500 = 300/500 = 0,6

μPsdBANYAK[300] = (300 − 100) / 500 = 200/500 = 0,4

Sanity check: Untuk himpunan yang berpasangan (TURUN/NAIK, SEDIKIT/BANYAK), jika satu turun satu naik, totalnya = 1. Di sini: 0,25 + 0,75 = 1 dan 0,6 + 0,4 = 1. Konsisten.

2

Aplikasi Fungsi Implikasi (MIN) — Hitung α-predikat Tiap Rule

Untuk setiap rule, hitung α = min dari semua anteseden, lalu "potong" kurva konsekuen di level α.

RuleAntesedenα-predikatAksi ke Konsekuen
[R1] PmtTURUN(0,25) AND PsdBANYAK(0,4) min(0,25; 0,4) = 0,25 Kurva BERKURANG dipotong di 0,25
[R2] PmtTURUN(0,25) AND PsdSEDIKIT(0,6) min(0,25; 0,6) = 0,25 Kurva BERKURANG dipotong di 0,25
[R3] PmtNAIK(0,75) AND PsdBANYAK(0,4) min(0,75; 0,4) = 0,4 Kurva BERTAMBAH dipotong di 0,4
[R4] PmtNAIK(0,75) AND PsdSEDIKIT(0,6) min(0,75; 0,6) = 0,6 Kurva BERTAMBAH dipotong di 0,6
Observasi Penting

Aturan R1, R2, R4 bertambah kuat (0,25 → 0,6) seiring berubahnya kondisi. Ini mencerminkan knowledge: "Permintaan NAIK + Persediaan SEDIKIT" paling butuh produksi tinggi.

3

Agregasi / Komposisi (MAX) — Gabungkan Semua Output

Gabungkan 4 output aturan jadi satu kurva komposit dengan MAX per region.

3a. MAX per Sisi Konsekuen

Sisi BERKURANG (R1, R2) = max(0,25; 0,25) = 0,25

Sisi BERTAMBAH (R3, R4) = max(0,4; 0,6) = 0,6

3b. Hitung Batas a1 dan a2 pada Kurva Komposisi

Fungsi keanggotaan kurva naik BERTAMBAH: μ(z) = (z − 2000) / 5000

Batas a1 (di μ = 0,25):
(a1 − 2000) / 5000 = 0,25
a1 − 2000 = 1250
a1 = 3250

Batas a2 (di μ = 0,6):
(a2 − 2000) / 5000 = 0,60
a2 − 2000 = 3000
a2 = 5000

3c. Bentuk Kurva Komposit (3 Daerah)

μ[z] = 0,25;                         untuk z ≤ 3250
μ[z] = (z − 2000) / 5000;      untuk 3250 ≤ z ≤ 5000
μ[z] = 0,6;                           untuk z ≥ 5000

Gambar 2: Kurva komposit hasil agregasi MAX. Tiga daerah: A1 (kiri datar), A2 (tengah naik), A3 (kanan datar).

4

Defuzzifikasi (Centroid / Center of Gravity) — Cari Nilai Crisp z*

Metode Centroid

Mencari nilai tegas z* dengan membagi total momen (M) dengan total luas daerah (A) dari kurva komposit.

z* = Total Momen (M) / Total Luas (A)

4a. Hitung Luas Tiap Daerah

Daerah A1 (Persegi Panjang, z: 0–3250, tinggi 0,25):
A1 = 3250 × 0,25 = 812,5

Daerah A2 (Trapesium, z: 3250–5000, tinggi 0,25 ke 0,6):
A2 = (0,25 + 0,6) × (5000 − 3250) / 2 = 0,85 × 1750 / 2 = 743,75

Daerah A3 (Persegi Panjang, z: 5000–7000, tinggi 0,6):
A3 = (7000 − 5000) × 0,6 = 2000 × 0,6 = 1200

Total Luas (A):
A = 812,5 + 743,75 + 1200 = 2756,25

4b. Hitung Momen Tiap Daerah (Integral)

Momen didefinisikan sebagai: Mi = ∫ μ(z) · z dz pada batas daerahnya.

Momen M1 (z: 0–3250, μ = 0,25 konstan):
M1 = ∫03250 0,25 · z dz = [0,125 · z2]03250
   = 0,125 × 32502 = 0,125 × 10.562.500 = 1.320.312,5
Momen M2 (z: 3250–5000, μ = (z-2000)/5000):
M2 = ∫32505000 (z−2000)/5000 · z dz = ∫ (0,0002z2 − 0,4z) dz
   = [0,000067 · z3 − 0,2 · z2]32505000
   = (0,000067 × 50003 − 0,2 × 50002) − (0,000067 × 32503 − 0,2 × 32502)
   = (8.375.000 − 5.000.000) − (2.299.984 − 2.112.500)
   = 3.375.000 − 187.484 = 3.187.515,625
Momen M3 (z: 5000–7000, μ = 0,6 konstan):
M3 = ∫50007000 0,6 · z dz = [0,3 · z2]50007000
   = 0,3 × (70002 − 50002) = 0,3 × (49.000.000 − 25.000.000)
   = 0,3 × 24.000.000 = 7.200.000

Total Momen (M):
M = 1.320.312,5 + 3.187.515,625 + 7.200.000 = 11.707.828,125

4c. Hitung Nilai Defuzzifikasi z*

z* = M / A
   = 11.707.828,125 / 2756,25
   = 4247,74 ≈ 4248
Kesimpulan Studi Kasus

Berdasarkan perhitungan FIS Mamdani, perusahaan sebaiknya memproduksi 4248 kemasan makanan kaleng ABC per hari, dengan kondisi Permintaan = 4000 dan Persediaan = 300.

Artinya: lebih banyak dari permintaan aktual (4000) karena persediaan sedikit (300) — perlu buffer untuk jaga-jaga. Logis!

4. Metode Defuzzifikasi Alternatif

Centroid adalah default Mamdani, tapi ada 4 metode lain yang juga umum:

MetodePrinsipKarakteristik
Centroid (COG) Titik pusat massa kurva Paling akurat, smooth, default Mamdani
Bisektor Nilai z yang membagi area jadi 2 sama besar Median area, robust terhadap outlier
MOM (Mean of Maxima) Rata-rata dari semua z dengan μ maksimum Untuk himpunan dengan puncak datar
LOM (Largest of Maxima) Nilai z terbesar dengan μ maksimum Konservatif (nilai terbesar)
SOM (Smallest of Maxima) Nilai z terkecil dengan μ maksimum Agresif (nilai terkecil)
Fokus UAS

Soal UAS umumnya hanya menggunakan Centroid (sesuai PDF). Cukup kuasai Centroid — tahu bahwa ada 4 metode lain & kapan mereka relevan.

5. Trade-off: Mamdani vs Sugeno

Mamdani (yang kita pakai)

  • Konsekuen: Himpunan fuzzy (mis: Produksi BERTAMBAH)
  • Defuzzifikasi: Centroid (integral)
  • Kelebihan: Intuitif, mudah diinterpretasi, cocok untuk expert knowledge
  • Kekurangan: Komputasi lebih berat (perlu integral)

Sugeno (alternatif, tak dibahas)

  • Konsekuen: Fungsi linear (mis: z = ax + by + c)
  • Defuzzifikasi: Weighted average (tanpa integral)
  • Kelebihan: Sangat cepat, cocok untuk real-time/optimasi
  • Kekurangan: Kurang intuitif, butuh data training untuk fitting parameter

6. Rangkuman: Workflow 4-Langkah Mamdani

Checklist Pengerjaan Soal Mamdani
  1. Fuzzifikasi — Hitung μ untuk semua himpunan input pada nilai crisp yang diberikan.
  2. Hitung α-predikat tiap rule — Gunakan MIN untuk AND, MAX untuk OR. Terapkan implikasi MIN ke konsekuen.
  3. Agregasi — Gabungkan output semua rule dengan MAX. Identifikasi batas-batas a1, a2, ... di mana μ berubah.
  4. Defuzzifikasi Centroid — Pecah jadi beberapa region geometris (persegi, trapesium, segitiga). Hitung A_i dan M_i per region. z* = ΣM_i / ΣA_i.

"Mamdani mengubah bahasa manusia (‘agak naik’, ‘sedikit turun’) menjadi angka produksi yang presisi." — Esensi FIS
