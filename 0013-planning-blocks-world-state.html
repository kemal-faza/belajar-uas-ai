Lesson 13: Planning, Blocks World & State (FOL)

Problem-solving terstruktur di AI — representasi dunia balok dengan First-Order Logic

1. Apa itu Planning?

Definisi: Planning (Perencanaan)

Planning adalah metode problem-solving terstruktur dalam AI yang memecah masalah besar menjadi sub-masalah kecil, menyelesaikan tiap sub-masalah bertahap, lalu mengintegrasikan solusinya menjadi satu rencana utuh — sembari tetap mengelola interaksi antar sub-masalah agar tidak saling membatalkan.

Planning bekerja berdasarkan 4 prinsip kognitif utama:

# Prinsip Penjelasan
1 Memecah Masalah Bagi masalah kompleks jadi sub-sub masalah kecil yang lebih sederhana.
2 Menyelesaikan Sub-Masalah Selesaikan masing-masing sub-masalah satu per satu, bertahap.
3 Menggabungkan Solusi Integrasikan solusi sub-masalah menjadi satu rencana utuh yang lengkap.
4 Menangani Interaksi Saat menggabungkan, kelola interaksi agar sub-masalah tidak saling membatalkan/menghancurkan.
Trade-off / Catatan UAS

Prinsip ke-4 (interaksi) adalah titik lemah utama algoritma planning klasik (lihat Lesson 15: Sussman Anomaly). Di soal UAS, jika ditanya "kelemahan GSP", jawabnya selalu terkait prinsip ke-4 ini: GSP tidak mengelola interaksi sub-goal secara global.

2. Blocks World (Dunia Balok)

Blocks World adalah domain studi kasus paling populer dalam perencanaan AI. Ini adalah tempat di mana robot memanipulasi balok di atas meja. Sederhana secara visual, tapi cukup kaya untuk menguji algoritma planning.

2.1 Komponen Lingkungan

Komponen Deskripsi
Meja (Table) Permukaan datar horizontal tempat meletakkan balok. Single surface.
Balok (Blocks) Sejumlah kubus identik ukuran & berat, biasanya dilabeli A, B, C, D, ...
Lengan Robot (Robot Arm) Single mechanical arm yang bisa mengambil, memindahkan, dan menumpuk balok. Hanya satu lengan — tidak bisa pegang 2 balok sekaligus.

2.2 Permasalahan Utama

Problem Statement Blocks World

Tentukan urutan aksi lengan robot untuk mengubah konfigurasi balok dari Initial State (keadaan awal) menjadi Goal State (keadaan akhir yang diinginkan) secara efisien.

Gambar 1. Visualisasi dunia balok. Lengan robot implicit (bisa mengambil 1 balok pada satu waktu).

Perhatikan: ini hanya salah satu konfigurasi. Planning akan mencari urutan langkah untuk mengubahnya ke Goal State (akan kita bahas di Lesson 15).

3. Representasi State dengan FOL

State dunia balok direpresentasikan secara formal menggunakan First-Order Logic (FOL). Ada 5 predikat dasar yang menangkap semua kondisi yang relevan.

3.1 Lima Predikat Dasar

Predikat Arti (Plain Language) Implikasi Logis
ONTABLE(x) Balok x berada langsung di atas meja. Jika ONTABLE(x) TRUE, maka x TIDAK dipegang (¬HOLDING(x)) dan TIDAK di atas balok lain (¬∃y ON(x,y)).
CLEAR(x) Bagian atas balok x kosong; tidak ada balok lain di atasnya. Jika CLEAR(x) TRUE, maka tidak ada y sedemikian sehingga ON(y,x) (tidak ada yang nempel di atas x).
ON(x, y) Balok x terletak langsung di atas balok y. Jika ON(x,y) TRUE, maka baik x maupun y TIDAK dipegang, dan tidak ada balok lain di atas x (kecuali tidak ada ON yang kontradiktif).
HOLDING(x) Lengan robot sedang memegang/mencengkeram balok x. Jika HOLDING(x) TRUE, maka lengan TIDAK empty (¬ARMEMPTY) dan x TIDAK di meja atau di atas balok lain.
ARMEMPTY Lengan robot dalam keadaan kosong; tidak memegang apa-apa. Jika ARMEMPTY TRUE, maka tidak ada x sedemikian sehingga HOLDING(x).
Implisit Penting

Jika ada 4 balok A, B, C, D di atas meja, maka implisitnya semua CLEAR (kecuali ada yang menumpuk). Biasanya CLEAR untuk balok yang di atas tidak ditulis eksplisit di Initial State, tapi selalu diasumsikan TRUE.

3.2 Rumus FOL Formal (Untuk Referensi)

Untuk UAS, sering diminta menuliskan rumus FOL formal. Berikut translasi langsung dari PDF:

∀x   ONTABLE(x) → ¬HOLDING(x) ∧ [¬∃y   ON(x,y)]
∀x   CLEAR(x) → ¬∃y   ON(y,x)
∀x [∃y   ON(x,y)] → ¬HOLDING(x) ∧ ¬HOLDING(y) ∧ ¬ON(y,x) ∧ [¬∃z   ON(x,z) ∨ ON(y,z)]
∃x   HOLDING(x) → ¬ARMEMPTY ∧ ¬ONTABLE(x) ∧ [¬∃y   ON(x,y) ∨ ON(y,x)]
ARMEMPTY → ¬∃x   HOLDING(x)

Baca: "∀x" = untuk semua x, "∃x" = ada x, "→" = implies, "∧" = AND, "∨" = OR, "¬" = NOT.

4. Contoh State Dunia Balok

Berikut dua contoh deskripsi state lengkap dalam FOL (langsung dari PDF):

Kasus [1]: Lengan Kosong, A di Meja

Kasus 1 — Skenario Sederhana

Kondisi dunia: lengan robot kosong, balok A diletakkan di atas meja.

ONTABLE(A) ∧ CLEAR(A) ∧ ARMEMPTY

CLEAR(A) ditulis eksplisit di sini karena hanya ada 1 balok dan ingin tegaskan "tidak ada apa-apa di atas A".

Kasus [2]: Lengan Memegang C, B di Atas A

Kasus 2 — Lebih Kompleks

Kondisi dunia: lengan robot memegang balok C. Di atas meja, balok B ditumpuk di atas balok A.

ONTABLE(A) ∧ ON(B,A) ∧ CLEAR(B) ∧ HOLDING(C) ∧ CLEAR(C)

Perhatikan: ARMEMPTY TIDAK ada (karena lengan pegang C). ONTABLE(C) TIDAK ada (karena C dipegang). CLEAR(B) TRUE karena tidak ada di atas B. CLEAR(C) TRUE karena bagian atas C kosong.

Jebakan Umum

Mahasiswa sering mencantumkan CLEAR untuk semua balok secara otomatis. Padahal CLEAR hanya TRUE untuk balok yang bagian atasnya benar-benar kosong. Pada Kasus 2 di atas, CLEAR(A) FALSE karena B di atas A — jadi CLEAR(A) tidak dicantumkan (implisit FALSE).

5. Quick Cheatsheet: 5 Predikat

Predikat Mnemonic Kapan TRUE
ONTABLE(x) on the table x langsung di meja, tidak di atas balok lain, tidak dipegang.
CLEAR(x) top is clear Tidak ada balok lain di atas x.
ON(x, y) x on top of y x langsung menempel di atas y.
HOLDING(x) arm is holding x Lengan sedang memegang x.
ARMEMPTY arm is empty Lengan tidak memegang apa-apa.
Hubungan Logis Antar Predikat
  • HOLDING(x) ⇔ ¬ARMEMPTY ⇔ ¬ONTABLE(x) ⇔ ¬∃y ON(x,y)
  • ARMEMPTY ⇔ ¬∃x HOLDING(x)
  • Sebuah balok bisa CLEAR tapi TIDAK ONTABLE (mis. B di atas A: CLEAR(B) TRUE, ONTABLE(B) FALSE).
