Sistem Pakar: Anatomi Pikiran Digital

Lesson 4 — Pengenalan Sistem Pakar (Expert Systems)

Tujuan Lesson Setelah lesson ini, kamu bisa: (1) Menjelaskan apa itu sistem pakar dan bedanya dengan program tradisional, (2) Menggambarkan arsitektur inti dan 4 aktor yang terlibat, (3) Menyebutkan DNA sistem pakar dan kapan tepat menggunakannya, (4) Memahami proses akuisisi pengetahuan dan tantangan tacit knowledge.

1. Apa Itu Sistem Pakar?

Sistem Pakar (Expert System) adalah alat pengambilan keputusan interaktif berbasis komputer (computer-based decision tool) yang dirancang untuk memecahkan masalah rumit yang biasanya membutuhkan keahlian manusia (non-algorithmic expertise). Sistem ini menggabungkan fakta, aturan heuristik, dan pengetahuan dari pakar manusia untuk menghasilkan rekomendasi.

Contoh Klasik
  • MYCIN (Stanford University) — Mendiagnosis infeksi bakteri dan meningitis, merekomendasikan dosis antibiotik. Ini adalah pionir sistem pakar medis.
  • Rice-Crop Doctor (National Institute of Agricultural Extension Management) — Mendiagnosis hama dan penyakit tanaman padi, memberi saran penanganan untuk petani.

2. Program Tradisional vs Sistem Pakar

Perbedaan fundamental terletak pada bagaimana logika dan pengetahuan diintegrasikan:

AspekProgram TradisionalSistem Pakar
Rumus Algoritma + Struktur Data Mesin Inferensi + Basis Pengetahuan
Logika Hardcoded menyatu dengan data Dipisahkan: cara berpikir vs apa yang diketahui
Perubahan Aturan Code harus dirombak total Cukup update Knowledge Base
Analogi Mesin kaku dengan satu fungsi Otak manusia: memori + logika terpisah
[Algoritma] + [Struktur Data] = Program Tradisional
vs
[Inference Engine] + [Knowledge Base] = Sistem Pakar

3. Arsitektur Inti (4 Komponen)

Sistem pakar terdiri dari empat komponen yang saling berinteraksi:

User Interface Antarmuka dialog interaktif
Inference Engine "Otak" sistem — mengolah data + aturan
Knowledge Base Kepakaran terenkode (aturan IF-THEN)
Working Storage Memori jangka pendek — data spesifik sesi konsultasi
  1. User Interface (UI) — Jembatan antara pengguna dan sistem. Pengguna memasukkan data/pertanyaan, sistem memberikan jawaban/rekomendasi.
  2. Inference Engine (Mesin Inferensi) — "Otak" sistem. Mengolah data dari Working Storage berdasarkan aturan di Knowledge Base. Juga menangani explanation (Why/How) dan uncertainty handling.
  3. Knowledge Base (Basis Pengetahuan) — Menyimpan kepakaran dari pakar, umumnya dalam bentuk aturan IF-THEN. Ini adalah memori jangka panjang sistem.
  4. Working Storage (Memori Kerja) — Memori jangka pendek yang menyimpan data spesifik dari sesi konsultasi yang sedang berlangsung.
ALIRAN DATA User memasukkan data via UI → Data disimpan di Working Storage → Inference Engine mencocokkan data dengan aturan di Knowledge Base → Hasil dikembalikan ke User via UI.

4. Para Aktor dalam Ekosistem Sistem Pakar

Empat peran penting yang terlibat dalam siklus hidup sistem pakar:

AktorPeranAktivitas Utama
Domain Expert Sumber pengetahuan Pakar manusia dengan intuisi dan pengalaman di bidangnya
Knowledge Engineer Penerjemah Mengekstrak pengetahuan dari pakar, merumuskan ke format logis untuk mesin
System Engineer Arsitek sistem Membangun kerangka software: inference engine, UI, infrastruktur
User Pengguna akhir Berkonsultasi dengan sistem untuk mendapat solusi/rekomendasi

Hubungan User dengan sistem disebut Interaction Loop.

5. DNA Sistem Pakar (4 Karakteristik Utama)

  1. Interaktif & Solutif — Memberikan respons aktif terhadap input dan menghasilkan rekomendasi spesifik.
  2. Inferensi Logis — Tidak sekadar menyimpan data pasif, tetapi merangkainya untuk menarik kesimpulan baru. Analogi: menghafal kosakata tanpa tahu cara merangkai kalimat tidak berguna. Menggunakan Forward Chaining atau Backward Chaining.
  3. Domain Spesifik — Pengetahuan sangat terfokus. Sistem pakar medis tidak bisa menangani masalah keuangan. Performa berbanding lurus dengan kualitas Knowledge Base.
  4. Nilai Keyakinan (Confidence Values) — Mampu menangani data tidak pasti/tidak lengkap dengan Certainty Factor (skor keyakinan, umumnya 0.0–1.0).

6. Kapan Menggunakan Sistem Pakar?

Kelebihan (Tepat Digunakan)Kekurangan (Hati-hati)
  • Masalah butuh pendekatan heuristik (intuisi pakar)
  • Cost-effective saat pakar manusia langka atau mahal
  • Pengetahuan perlu didokumentasikan dan direproduksi
  • Tidak cocok untuk masalah numerik murni (cukup rumus matematika)
  • Biaya awal mahal (knowledge engineering)
  • Tidak 100% bebas error — rentan kesalahan interpretasi

7. Akuisisi Pengetahuan: Tahap Paling Krusial

Proses mentransfer keahlian dari pakar ke sistem komputer. Inilah tahap paling sulit dalam pembangunan sistem pakar.

5 Tahap Akuisisi Pengetahuan:

  1. Elicitation (Penggalian) — Menggali pengetahuan dari pakar via wawancara, kuesioner, observasi.
  2. Collection (Pengumpulan) — Mengumpulkan dokumen, buku teks, laporan studi kasus.
  3. Analysis (Analisis Pola) — Memetakan alur penalaran pakar saat mengambil keputusan.
  4. Modelling (Pemodelan Logika) — Merepresentasikan pola ke model logika (decision tree, aturan formal).
  5. Validation (Validasi) — Menguji model bersama pakar untuk memastikan tidak ada salah tafsir.
Tantangan Terbesar: TACIT KNOWLEDGE Gunung es pengetahuan: Eksplisit (teori, SOP, rumus) hanyalah ujung gunung es yang muncul. Bagian terbesar adalah Tacit Knowledge — insting, intuisi, dan pengalaman bertahun-tahun yang bahkan sang pakar sendiri tidak sadar menggunakannya. Ini sangat sulit diekstraksi.
