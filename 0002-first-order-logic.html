First-Order Logic (FOL)

Lesson 2 — Predicate Logic & Quantifiers untuk UAS

Prerequisites: Kamu sudah paham Propositional Logic (syntax, semantics, inference rules, Horn Form, Resolusi) dari Lesson 1.

1. Kenapa FOL? тАФ Keterbatasan Propositional Logic

Propositional Logic punya kelemahan besar: tidak bisa merepresentasikan objek, relasi, dan kuantitas.

Contoh:

Di Wumpus World, kita ingin bilang: "Semua kotak yang berbau Stench pasti memiliki Wumpus di kotak sebelahnya."

Di PL, kita harus menulis satu rule untuk SETIAP kotak:

StenchтВБ,тВБ тЗТ WтВБ,тВВ тИи WтВВ,тВБ
StenchтВБ,тВВ тЗТ WтВБ,тВБ тИи WтВБ,тВГ тИи WтВВ,тВВ
StenchтВВ,тВБ тЗТ WтВБ,тВБ тИи WтВВ,тВВ тИи WтВГ,тВБ
...

Bayangkan untuk grid 4├Ч4 = 16 kotak, kita butuh 16 rules! Untuk grid 100├Ч100 = 10.000 rules! Tidak praktis.

Dengan FOL, cukup SATU rule:

тИАx,y Stench(x,y) тЗТ (Wumpus(x+1,y) тИи Wumpus(x-1,y) тИи Wumpus(x,y+1) тИи Wumpus(x,y-1))

Ini kenapa FOL disebut first-order тАФ kita bisa "berbicara tentang" objek-objek dan relasinya, bukan cuma proposisi utuh.

PL vs FOL: Di PL, "WтВБ,тВГ" adalah proposisi (True/False). Di FOL, "Wumpus(1,3)" adalah predikat yang menerima argumen тАФ kita bisa query, substitusi, dan kuantifikasi.

2. Syntax FOL

Komponen Dasar

KategoriDefinisiContoh
Constant Objek spesifik di dunia 1, 2, 3, gold, wumpus
Variable Objek yang belum dispesifikkan x, y, z, a, b
Predicate Relasi / properti dari objek Stench(x,y), Pit(x), Adjacent(a,b)
Function Memetakan objek ke objek lain adjacent_to(x), next_cell(x,y)
Quantifier Kuantitas objek yang memenuhi тИА (forall), тИГ (exists)
Connective Sama seperti di PL ┬м, тИз, тИи, тЗТ, тЗФ
Equality Kesamaan dua term =(x, y) atau x = y

Term vs Sentence

Term = ekspresi yang merepresentasikan objek:

  • Constant: wumpus
  • Variable: x
  • Function: father(john), plus(2,3)

Sentence (Kalimat) = ekspresi yang bernilai True/False:

  • Atomic sentence: Pit(3,1), Stench(1,2), Adjacent(2,1, 2,2)
  • Complex sentence: Stench(1,2) тЗТ Wumpus(1,3)
  • Quantified sentence: тИАx Pit(x) тЗТ Breeze(adjacent(x))

3. Quantifiers (Kuantifier)

A. Universal Quantification (тИА)

тИАтЯиvariableтЯй тЯиsentenceтЯй

Baca: "Untuk semua nilai variable, sentence bernilai True."

Contoh:

тИАx Pit(x) тЗТ Breeze(adjacent(x))
"Semua kotak yang punya Pit pasti ada Breeze di kotak sebelahnya."

тИАx,y Adjacent(x,y) тЗТ Adjacent(y,x)
"Jika x bertetangga dengan y, maka y juga bertetangga dengan x." (simetris)
тЪая╕П Hati-hati dengan тЗТ dan тИз di bawah тИА:
тИАx Pit(x) тЗТ Breeze(x) тАФ benar, artinya "semua Pit menghasilkan Breeze"
тИАx Pit(x) тИз Breeze(x) тАФ artinya "semua objek adalah Pit DAN ber-Breeze" тЭМ (salah total untuk domain yang punya objek non-Pit)

Rule of thumb: тИА biasanya pakai тЗТ.

B. Existential Quantification (тИГ)

тИГтЯиvariableтЯй тЯиsentenceтЯй

Baca: "Ada setidaknya satu nilai variable yang membuat sentence True."

Contoh:

тИГx Pit(x) тИз Breeze(x)
"Ada sebuah kotak yang punya Pit dan ber-Breeze."

тИГx Gold(x) тИз Glitter(x)
"Ada sebuah kotak berisi emas yang mengkilau."
тЪая╕П Hati-hati dengan тЗТ dan тИз di bawah тИГ:
тИГx Pit(x) тИз Breeze(x) тАФ benar, artinya "ada Pit yang ber-Breeze"
тИГx Pit(x) тЗТ Breeze(x) тАФ berbahaya! Ini akan True jika ada satu saja objek yang BUKAN Pit (karena implication PтЗТQ True saat P=False). Hampir selalu salah digunakan.

Rule of thumb: тИГ biasanya pakai тИз.

C. Nested Quantifiers

EkspresiMakna
тИАx тИАy Loves(x,y)Semua orang mencintai semua orang
тИАx тИГy Loves(x,y)Setiap orang mencintai seseorang
тИГx тИАy Loves(x,y)Ada seseorang yang dicintai semua orang
тИГx тИГy Loves(x,y)Ada seseorang yang mencintai seseorang

Urutan quantifier PENTING! тИАx тИГy тЙа тИГy тИАx.

4. Semantics FOL

Semantics FOL mendefinisikan makna kalimat dalam suatu domain (himpunan objek).

  • Domain: himpunan objek yang ada di dunia (misal: semua kotak di grid 4├Ч4)
  • Interpretation: mapping dari:
    • Constant тЖТ objek spesifik di domain
    • Predicate тЖТ himpunan objek/relasi
    • Function тЖТ fungsi dari objek ke objek
  • Model: sebuah interpretation + domain yang membuat semua kalimat di KB True

Entailment di FOL: Sama seperti PL тАФ KB тКи ╬▒ jika ╬▒ True di semua model dimana KB True. Tapi model di FOL bisa tak terhingga (domain infinite), jadi lebih kompleks.

5. Inference di FOL

A. Substitution (Substitusi)

Mengganti variable dengan term:

SUBST({x/3, y/1}, Stench(x,y)) = Stench(3,1)

B. Universal Instantiation

Dari тИАx kita bisa turunkan untuk constant tertentu:

тИАx Stench(x) тЗТ Wumpus(adjacent(x))
тЖТ Stench(1,2) тЗТ Wumpus(adjacent(1,2))

C. Existential Instantiation

Dari тИГx kita bisa ganti dengan constant baru (Skolem constant):

тИГx Pit(x) тИз Breeze(x)
тЖТ Pit(SтВБ) тИз Breeze(SтВБ)   // SтВБ adalah constant baru yang belum dipakai

D. Unification (Unifikasi)

Mencari substitusi yang membuat dua kalimat identik. Ini inti dari inferensi FOL.

Kalimat 1Kalimat 2Unifier (╬╕)
Stench(x,1)Stench(2,y){x/2, y/1}
Pit(x)Pit(3){x/3}
Knows(x, y)Knows(John, z){x/John, y/z}
Knows(x, x)Knows(John, Mary)FAIL (x tidak bisa = John dan Mary)

Ini adalah Modus Ponens versi FOL тАФ untuk Horn clauses dengan quantifier:

6. Wumpus World dalam FOL

Sekarang kita representasikan Wumpus World dengan FOL тАФ jauh lebih ringkas daripada PL.

Predicates yang kita butuhkan:

Wumpus(x,y)       тАФ Ada Wumpus di [x,y]
Pit(x,y)          тАФ Ada Pit di [x,y]
Breeze(x,y)       тАФ Ada Breeze di [x,y]
Stench(x,y)       тАФ Ada Stench di [x,y]
Glitter(x,y)      тАФ Ada kilauan emas di [x,y]
Gold(x,y)         тАФ Ada emas di [x,y]

Adjacent(x,y, a,b) тАФ [x,y] bertetangga dengan [a,b]
Safe(x,y)         тАФ [x,y] aman untuk dikunjungi

Pengetahuan Dunia (Domain Knowledge):

// Aturan Breeze: Breeze ada jika dan hanya jika ada Pit di kotak tetangga
тИАx,y Breeze(x,y) тЗФ тИГa,b Adjacent(x,y, a,b) тИз Pit(a,b)

// Aturan Stench: Stench ada jika dan hanya jika ada Wumpus di kotak tetangga
тИАx,y Stench(x,y) тЗФ тИГa,b Adjacent(x,y, a,b) тИз Wumpus(a,b)

// Aturan Glitter: Glitter ada jika ada Emas
тИАx,y Glitter(x,y) тЗФ Gold(x,y)

// Aturan Aman: kotak aman jika tidak ada Pit dan tidak ada Wumpus
тИАx,y Safe(x,y) тЗФ ┬мPit(x,y) тИз ┬мWumpus(x,y)

// Adjacency: definisi tetangga
тИАx,y,a,b Adjacent(x,y, a,b) тЗФ
  (a = x+1 тИз b = y) тИи (a = x-1 тИз b = y) тИи
  (a = x тИз b = y+1) тИи (a = x тИз b = y-1)

Fakta dari Percept:

// Agen bergerak dari [1,1] ke [2,1]
┬мBreeze(1,1)         // tidak ada Breeze di start
┬мStench(1,1)         // tidak ada Stench di start

Breeze(2,1)          // ada Breeze di [2,1]
┬мStench(2,1)         // tidak ada Stench di [2,1]

Inferensi FOL dengan Forward Chaining:

// Dari Breeze(2,1) + rule Breeze:
тИГa,b Adjacent(2,1, a,b) тИз Pit(a,b)

// Adjacent [2,1] dengan: [1,1], [2,2], [3,1]
Pit(1,1) тИи Pit(2,2) тИи Pit(3,1)

// Dari fakta ┬мPit(1,1) (start aman):
Pit(2,2) тИи Pit(3,1)

// Dari ┬мBreeze(1,1) тАФ tidak ada Breeze di [1,1]:
// Berarti tidak ada Pit di tetangga [1,1] yaitu [1,2] dan [2,1]
// Sudah diketahui [2,1] tidak ada Pit (aman)
// Jadi Pit tidak mungkin di [2,2]? Tunggu...

Sebenernya dari ┬мBreeze(1,1), karena satu-satunya tetangga [1,1] yang belum kita ketahui adalah [1,2] dan [2,1], dan [2,1] aman... kita tahu tidak ada Pit di [2,2]? Tidak juga. Adjacent(1,1, 2,2) itu False karena [1,1] dan [2,2] diagonal, bukan tetangga.

Jadi dari ┬мBreeze(1,1) kita simpulkan: ┬мPit(1,2) тИз ┬мPit(2,1)

Dan dari Breeze(2,1): pit di [1,1] (sudah aman), [2,2], atau [3,1]

Maka: Pit di [2,2] atau [3,1] тАФ sama persis dengan hasil di PL!
